# xhs_analysis

一个面向小红书内容创作场景的本地分析与提示词生成工具。

## 阶段总结

### 1. 什么痛点

- 用户拿到一条笔记链接后，往往要手动拆标题、正文、标签、图片链接，再整理成仿写和生图提示词，过程重复且耗时。
- AI 能力虽然已经接入，但授权 key 没有真正贯穿到完整流程里，导致“能解析、不能生成 prompt”的体验割裂。
- 参考图分析一旦超时，整个 prompt 生成会直接失败，流程稳定性不够。
- 仓库里存在部分历史 demo / 辅助脚本文件，容易增加维护成本和判断噪音。

### 2. 解决了什么问题

- 统一了“输入链接 -> 解析内容 -> 生成仿写 prompt -> 生成生图 prompt -> 自测”的主流程。
- 增加了授权 key 输入与保存能力，并让 key 真正参与 `parse`、`prompts`、`selftest` 全链路。
- 优化了异常处理逻辑：参考图分析失败时不再直接中断，而是降级继续生成 prompt。
- 清理了无关分支流程中的冗余内容，并将当前阶段成果沉淀为仓库说明，便于后续接手和发布。

### 3. 怎么使用

1. 在本地启动服务：

```bash
node server.js
```

2. 在浏览器打开本地页面，输入小红书笔记链接。

3. 在“授权key”输入框中填入可用的 Bearer token，点击“保存”。

4. 点击“开始解析”，等待系统提取标题、正文、标签和媒体内容。

5. 在生成模块中继续生成仿写 prompt、生图 prompt，并按需执行自测。

## 当前主流程

- 输入笔记链接
- 输入并保存授权 key
- 解析笔记内容
- 生成仿写 prompt
- 生成生图 prompt
- 执行自测

## 启动说明

- 运行环境：Node.js
- 启动命令：`node server.js`
- 默认入口：浏览器访问本地服务地址
